# Supabase RSVP Setup

Dokumen ini dipakai agar data RSVP tersimpan online, bisa dibuka dari mana saja, dan jumlah `hadir`, `tidak hadir`, serta doa tamu tampil sinkron di website.

## 1. Isi Environment Variable

Tambahkan env berikut ke `.env.local` untuk lokal dan ke Vercel untuk production:

```bash
SUPABASE_URL=https://YOUR_PROJECT.supabase.co
SUPABASE_SERVICE_ROLE_KEY=YOUR_SERVICE_ROLE_KEY
```

`SUPABASE_ANON_KEY` bersifat opsional. Kode saat ini memprioritaskan `SUPABASE_SERVICE_ROLE_KEY` agar route server bisa membaca dan menulis RSVP tanpa policy publik tambahan.

## 2. Buat Tabel RSVP

Jalankan SQL ini di `Supabase SQL Editor`:

```sql
create table if not exists public.rsvp_entries (
  id bigint generated by default as identity primary key,
  name text not null,
  attendance text not null check (attendance in ('hadir', 'tidakHadir')),
  message text not null default '',
  created_at timestamptz not null default timezone('utc', now())
);
```

## 3. Jika Ingin Mengaktifkan RLS

Kalau Anda mengaktifkan Row Level Security, route server ini tetap aman selama memakai `SUPABASE_SERVICE_ROLE_KEY`.

Opsional:

```sql
alter table public.rsvp_entries enable row level security;
```

## 4. Cara Kerja Sekarang

- Website akan membaca data RSVP langsung dari Supabase saat env tersedia.
- Jika Supabase belum diisi saat development lokal, aplikasi akan fallback ke `.data/metatah-rsvp.json`.
- Saat RSVP berhasil dikirim, browser diberi cookie agar tombol `Kirim RSVP` hanya bisa dipakai sekali per perangkat/browser.
- Daftar doa sekarang ikut menampilkan status `Hadir` atau `Tidak Hadir`.

## 5. Verifikasi

Setelah env diisi:

```bash
npm run lint
npm run build
```

Lalu buka website, kirim satu RSVP, dan refresh halaman. Tombol kirim harus tetap nonaktif dan data RSVP harus tetap muncul.
